# Human-in-the-Loop Gate

The final component before any output leaves the system. Presents the
Daily Brief to the school counselor and requires explicit human approval
before any referral record is written. No automated action is taken on
student wellbeing data without counselor sign-off.

---

## Role

Enforce the human decision boundary. The pipeline produces recommendations;
the counselor makes decisions. The HITL gate is the structural mechanism
that enforces this separation unconditionally.

---

## Trigger condition

The Orchestrator invokes the HITL gate when:

1. Daily Brief has been assembled for all students in the batch
2. LLM-as-judge score >= 0.75 (pass threshold)

If judge score < 0.75, the Orchestrator presents the failure reason
alongside the brief so the counselor can see the quality concern.

---

## What the counselor sees

```
DAILY BRIEF — [Date]
Generated by: Mental Health First Responder Agent

URGENT ACTION REQUIRED ([n] students):
  • [student_id] — Pattern break detected. Sudden drop from baseline.
    Text response flagged for distress keywords.
    Recommend: direct check-in today.

  • [student_id] — [n] consecutive low-mood days.
    Recommend: one-on-one conversation + parent notification consideration.

ELEVATED WATCH ([n] students):
  • [student_id] — Declining trend over 2 days. Monitor.

ROUTINE ([n] students): No action required today.

─────────────────────────────────────────
Evaluation Score: [judge_score] / 1.0
─────────────────────────────────────────

[ APPROVE AND LOG ]   [ REQUEST MORE CONTEXT ]   [ OVERRIDE — NO ACTION ]
```

**PII guarantee:** No real student names appear in the Daily Brief.
Student IDs only throughout. The counselor maps IDs to students via
their own records outside the system.

---

## Counselor action options

```yaml
actions:
  APPROVE_AND_LOG:
    outcome: referral log entries written for all urgent and elevated students
    referral_log_entry:
      student_id: string
      date: ISO8601
      priority: enum [urgent, elevated]
      recommended_action: string
      counselor_approved_at: ISO8601

  REQUEST_MORE_CONTEXT:
    outcome: Orchestrator re-invokes Memory Keeper in read-only mode for
             the named student; returns full 7-day signal history summary
             to counselor; does not re-run full pipeline
    note: counselor may request context for multiple students sequentially;
          APPROVE or OVERRIDE must still be selected to close the session

  OVERRIDE_NO_ACTION:
    outcome: no referral records written
    audit_log_entry:
      run_date: ISO8601
      override_reason: optional free text from counselor
      overridden_at: ISO8601
    note: override is always logged for audit trail; it cannot be silent
```

---

## Hard rules

```yaml
hitl_rules:
  no_referral_without_approval: true
    # referral_log is never written until APPROVE AND LOG is selected
    # this rule has no exceptions — not for urgent students, not for crisis_watch

  no_timeout_auto_approve: true
    # if counselor does not respond, session remains pending
    # no auto-approval after any time threshold

  override_always_logged: true
    # OVERRIDE outcome writes an audit log entry regardless of reason given

  student_name_never_in_brief: true
    # brief content is audited for PII before presentation (part of judge rubric)
```

---

## Relationship to LLM-as-judge

The HITL gate receives the judge scorecard alongside the Daily Brief.
The scorecard is surfaced to the counselor as a quality indicator —
not to block their decision, but to give them appropriate context about
pipeline confidence.

```yaml
judge_scorecard_display:
  overall_score: float          # weighted composite
  criteria_breakdown:
    signal_coverage: score
    escalation_accuracy: score
    pii_free_output: pass/fail
    counselor_action_clarity: score
    false_positive_rate: score
```

---

## Audit trail

Every session produces an immutable audit record regardless of outcome:

```yaml
audit_record:
  run_date: ISO8601
  students_processed: int
  students_skipped: int
  urgent_count: int
  elevated_count: int
  routine_count: int
  judge_score: float
  hitl_outcome: enum [approved, overridden, pending]
  referrals_logged: int
  counselor_action_at: ISO8601 | null
```
